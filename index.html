📖 Documentation — Comment ça marche ? ▼
⚠️ Avertissement — Cet outil est une aide à la préparation, pas un substitut au manuel de vol. Le pilote commandant de bord est seul responsable de ses décisions opérationnelles.

1. Comment fonctionne le calcul ?

Le calculateur lit les tables de performance du manuel de vol de chaque avion (AFM) et interpole entre les valeurs pour les conditions exactes du vol.

Trois paramètres sont interpolés simultanément :

  • Altitude-densité (élévation terrain en ft)
  • Température OAT (°C)
  • Masse (kg) — pour les DR400 uniquement

La distance calculée est toujours la distance de passage des 15 m (obstacle clearance), conformément à l'AFM. Le roulement est affiché à titre informatif.

2. Profils avion disponibles

  • DR400 — F-GCAL, F-BXEK (120 cv, hélice Sensenich) : tables 3 altitudes × 3 températures × 2 masses (700 / 900 kg). Tables séparées piste dure et herbe.
  • DR300 — F-BTBS, F-BTXS, F-BTKG (120 cv, hélice Sensenich) : tables 4 altitudes × 4 températures. La masse est intégrée via un facteur d'échelle linéaire.

Pour ajouter un avion, il suffit d'ajouter son fichier JSON dans /aircraft/ dans le dépôt GitHub — aucune modification du code nécessaire.

3. Marge de sécurité

Une marge multiplicative est appliquée sur la distance calculée (30 % par défaut, réglable dans la barre en haut) :

Requis = Distance calculée × (1 + marge / 100)

C'est cette valeur «Requis, marge incluse» qu'il faut comparer à la TODA ou LDA disponible. Si Requis ≤ TODA/LDA → OK. Sinon → refus.

4. Vent — composantes et facteurs

Les composantes sont calculées trigonométriquement à partir du QFU et de la direction/vitesse vent :

  • Face = V × cos(angle QFU−vent)
  • Arrière = max(0, −face)
  • Travers = |V × sin(angle)|

Le facteur vent est ensuite appliqué à la distance de base :

  • Vent de face : interpolation linéaire entre 0 kt (×1,00), 10 kt (×0,78), 20 kt (×0,63), 30 kt (×0,52)
  • Vent arrière : +10 % par tranche de 2 kt (ex. 4 kt arrière → ×1,20)

Limite travers : Alerte à 18 kt. Calcul bloqué à 22 kt (limite avion).

5. Vent variable (VRB)

Avec secteur (ex. 340V040 dans le METAR) : balayage degré par degré dans l'arc, vitesse moy. ET rafales — la composante la plus défavorable est retenue.

VRB sans secteur (ex. VRB03KT) : direction inconnue → vent arrière pur appliqué. C'est l'hypothèse la plus conservative. Signalé dans les résultats.

6. Vent réglementaire EASA (50 % / 150 %)

Option activable dans la barre de paramètres. Conforme CAT.POL.A.305 / CAT.POL.A.205 :

  • Vent de face : seuls 50 % de la composante sont créditée (conservateur)
  • Vent arrière : 150 % sont appliqués (conservateur)

7. Option «5 kt arrière»

S'affiche quand le vent calculé est moins défavorable qu'un vent arrière de 5 kt. Permet d'appliquer cette hypothèse conservatrice utilisée en opérations pour couvrir les erreurs de virement de bord ou les calmes trompeurs.

8. METAR et pistes automatiques

Le bouton METAR charge automatiquement : vent (direction, vitesse, rafales, VRB), OAT et élévation. Si la station exacte est indisponible, la plus proche est utilisée. Un groupe TEMPO avec vent plus défavorable remplace le vent principal.

Les pistes disponibles sont chargées via OpenAIP (QFU magnétique exact + TODA/LDA déclarées). Un clic sur une piste remplit les champs correspondants et remet les calculs à zéro.

9. TAF arrivée — Sélection de période

Le bouton TAF charge le TAF, les pistes et l'élévation en parallèle. Sélectionnez votre ETA en UTC pour voir quelles périodes TAF sont actives à votre heure d'arrivée.

Code couleur des périodes :

  • ⚠ Retenir — Période active à l'ETA avec la distance LDG la plus restrictive. C'est celle-ci à utiliser.
  • ✓ ETA — Période également active mais moins restrictive.
  • TEMPO — Période temporaire active. Incluse dans la comparaison.
  • Grisé — Période non active à l'ETA sélectionné.

Règles OMA (AMC1 CAT.POL.A.335 / tableau 4.13.1) :

  • PROB40 seul → inclus dans la comparaison (probabilité suffisante)
  • PROB30 seul → ignoré, affiché en pointillé gris (probabilité trop faible)
  • PROB30 TEMPO et PROB40 TEMPO → ignorés pour les performances (phénomène passager, non fiable)

10. Comparaison METAR / TAF

Si l'ETA tombe dans les 2 heures suivant l'émission du METAR, les deux sources sont comparées. La comparaison porte sur la vraie distance LDG calculée avec chaque source :

  • Si le METAR donne une distance plus longue → vent METAR appliqué automatiquement aux champs arrivée.
  • Si le TAF est plus restrictif → les champs restent sur le TAF, un message l'indique.

Cliquer sur une période TAF réapplique le vent TAF (override manuel).

11. Surface piste et état mouillé

Les facteurs de dégradation pour piste en herbe sont issus du tableau 1.18 (décollage) et 1.25.1 (atterrissage) de l'OMA Part C :

Condition Facteur Application
Herbe sèche×1,00Déco + Att.
Herbe mouillée×1,15Déco + Att.
Herbe courte mouillée×1,60Att. sans freins uniquement

Aucun facteur spécifique n'est prévu par l'AFM pour une piste dure mouillée au décollage — la marge de 30 % couvre cette variabilité.

12. Export PDF

Le bouton 📄 PDF génère une fiche vectorielle incluant : paramètres, résultats, détail des facteurs, METAR brut, TAF brut (période retenue surlignée en bleu), source météo retenue, heure du coucher du soleil et de la nuit aéronautique (coucher + 30 min) calculées depuis les coordonnées de la station.
